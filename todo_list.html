GMIS

GMIS任务


GMIS将立足实用原则迭代发展，直至实现自主编程。


初期，我们的目标是让普通人能使用自己的母语方便简单的实现逻辑编程，为此需要持续提高GMIS的用户体验，生成更多的外部物体，实现更多功能。

为了扩大影响力，我们将尽快演示任意数量的GMIS组成物联网，通过逻辑分工，实现超级计算机的计算能力。

以及实现手机客户端，让每个人的手机可以和自家里的PC实现远程相互调用，今儿实现控制智能家居的能力。

为此，我们需要在以下几个方面努力。


外部物体使用方面

 
1) 封装更多外部物体

目前成熟的C++第三方库已经非常多，同时很多常见网络公司都对外提供API，我们只需要简单封装，即可被GMIS逻辑调用。目前需要的封装如下：
 电子邮件
 网络操作
 web浏览
 解压缩
 PDF
 文件传输
 window API
 C函数库
 目录和文件操作
 加密解密
 字符串操作
 音乐播放
 网盘API访问
 社交网站API访问
 其他用户优先提到的功能

2）实现对非其他语言公开库的封装
除了众多的C\C++开放源码库，还有难以计数的其他语言编程结果，比如java,python等语言写的各种功能库，GMIS同样可以封装并利用。

3）更便利的执行外部物体执行
让GMIS能记忆所有使用过的外部物体，可以根据用户的需求，提示特定的外部物体，并知道在哪里可以找到，以及可以向其他GMIS寻求下载，或者直接叫其他GMIS代为执行。


建设虚拟空间

1 尝试实现GUI库的跨平台
2 完善3D空间的使用体验和视觉效果，
3 建设公共空间数据库, 实现GMIS之间互联互通，所有GMIS的虚拟空间和公共空间组成一个3D虚拟世界，实现无缝漫游，相同空间的GMIS之间可以聊天或请求对方执行逻辑任务

GMIS（Brain）

1 更多测试，使得Brain在任何情况下都能保证良好
2 更好的语言理解，能根据逻辑的上下文解决歧义
3 向linux和各手机平台迁移
4 优化用户体验，更好的交互、输出和提示，实现各语言版本
5 实现一个公共的GMIS作为母体，可以记忆任何信息，全功能，可以响应任何个人GMIS的请求，帮助用户教授自己的GMIS
